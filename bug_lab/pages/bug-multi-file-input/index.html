Bug Lab — Multi File Input: Selector Ambiguity

Bug: When a page has multiple <input type="file"> elements, the recorder/normalizer may generate identical or ambiguous selectors. If the healing system falls back to input[type=file], it matches ALL file inputs — causing strict mode violation.

Each input below uses a different identification strategy (label, aria-label, data-testid, id+name only). Tests verify that each generates a distinct, unique selector.
