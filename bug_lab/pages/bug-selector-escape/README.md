# BUG: Selector Escape — Quotes in Text/Attributes

## Symptom

CSS attribute selectors generated by `SelectorAgent` fail when
attribute values contain single quotes (`'`). The delimiter character
matches the quote inside the value, breaking the selector syntax.

Example:
```
[aria-label='Can't touch this']  ← broken, ' terminates string early
[data-testid='it's-a-test']      ← broken
```

`text=` selector handles double quotes via `\"` escaping but does
NOT handle the single-quote-in-value case for CSS attribute selectors.

## Cause

`SelectorAgent._try_testid()`, `_try_aria()`, `_try_placeholder()` build
CSS attribute selectors with hardcoded single-quote delimiters:

```python
# _try_aria line 98
new_locator=f"[aria-label='{label}']"   # ← label may contain '

# _try_placeholder line 113
new_locator=f"[placeholder='{ph}']"     # ← ph may contain '
```

No escaping of quote characters in values.

## Reproduction

```bash
# Start test server
cd bug_lab/pages && python -m http.server 8700

# Open browser
open http://localhost:8700/bug-selector-escape/index.html

# Try Playwright:
page.locator("[aria-label='Can\\'t touch this']").click()  # fails
page.locator("[data-testid='it\\'s-a-test']").click()      # fails
```

## Validation

```bash
pytest bug_lab/tests/test_bug_selector_escape.py -v
```

All unit tests pass. Slow browser tests reveal the bug:
- `test_aria_label_single_quote_fails` — passes (bug confirmed)
- `test_testid_single_quote_fails` — passes (bug confirmed)
- Workaround tests pass (get_by_role, ID fallback)
